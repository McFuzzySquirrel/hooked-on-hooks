# ADR-004: Isolate Visualizer Hooks in a Dedicated Subdirectory

- Status: Accepted
- Date: 2026-04-14

## Context

Bootstrap generated stub hook scripts and the `visualizer-hooks.json` manifest
directly in `.github/hooks/`. This co-mingled visualizer-generated files with
user-managed hooks and other integration manifests, creating several problems:

1. **Filename collisions.** Generated stubs like `session-start.sh` could
   overwrite or shadow user hooks with the same name. The `--prefix` flag
   mitigated this, but it was opt-in.
2. **Unclear ownership.** With no directory boundary, it was hard to tell which
   files were auto-generated by the visualizer and which were manually authored.
3. **Messy cleanup.** Unbootstrap had to inspect individual files to decide
   which to delete, rather than removing a single directory.
4. **Empty-manifest bug.** When `.github/hooks/` contained unrelated `.sh` files
   that did not match any event type, bootstrap created a manifest with
   `hooks: {}` — confusing and useless.

## Decision

Place all visualizer-generated files — stub hook scripts and the canonical
manifest — in a dedicated `.github/hooks/visualizer/` subdirectory.

1. **`--create-hooks` stubs** are written to `.github/hooks/visualizer/`.
2. **`visualizer-hooks.json`** is written to `.github/hooks/visualizer/`.
3. **Manifest command paths** reference `./.github/hooks/visualizer/<name>.sh`.
4. **Existing user hooks** elsewhere in `.github/hooks/` are still discovered
   and wired via the recursive scan — they stay in place.
5. **Unbootstrap** deletes the `visualizer/` subdirectory when empty after
   removing generated stubs and the manifest.
6. **Stub REPO_ROOT** traverses three levels up (`../../..`) instead of two,
   matching the deeper directory nesting.

## Rationale

1. Eliminates filename collisions without requiring `--prefix`.
2. Makes ownership obvious — everything in `visualizer/` is auto-generated.
3. Simplifies cleanup — unbootstrap can delete the whole subdirectory.
4. Fixes the empty-manifest bug: when no events are covered, the manifest is
   simply not created (guarded by an early return).

## Consequences

### Positive

1. Visualizer files are clearly separated from user-managed hooks.
2. `--prefix` becomes optional for collision avoidance (still supported for
   backward compatibility or additional namespacing).
3. Cleanup is a single directory removal rather than per-file inspection.
4. Code review diffs clearly show which files are visualizer-owned.

### Negative

1. Repos bootstrapped before this change have stubs in `.github/hooks/`
   (not in the subdirectory). Unbootstrap handles both legacy and new
   locations, but users should re-bootstrap to consolidate.
2. The REPO_ROOT depth in stub scripts changed from `../..` to `../../..`.
   `isAutoGeneratedStubContent` accepts both patterns for backward
   compatibility.

## Alternatives Considered

### A) Keep stubs in `.github/hooks/` and rely on `--prefix` for isolation

Rejected because `--prefix` is opt-in and doesn't address the ownership
visibility or cleanup problems.

### B) Use a configurable subdirectory name

Rejected as over-engineering. A fixed `visualizer/` name is clear,
predictable, and matches the project name.

## Follow-Up Actions

1. Update refresh docs to recommend unbootstrap + re-bootstrap for repos
   with legacy hook placement.
2. Consider whether the `--prefix` flag should be deprecated in a future
   release, given that the subdirectory already provides isolation.
